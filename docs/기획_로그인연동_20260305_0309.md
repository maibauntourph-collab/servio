# K-Barber 옴니채널 로그인 기획안 (Google, Facebook, Email)

안녕하세요 교수님! 제안해주신 다중 로그인 시스템(이메일, 구글, 페이스북) 도입은 사용자(학생 및 고객)의 접근성을 극대화하는 매우 중요한 단계입니다. 

코딩에 들어가기 앞서, 현재 우리의 기술 스택(React + Cloudflare Workers + NeonDB)에 가장 적합하고 안정적인 아키텍처와 UI/UX 방향성에 대해 정리하여 기획안을 제안해 드립니다. 학생들에게 설명하시기 좋도록 명확하고 이해하기 쉽게 구성했습니다.

---

## 🏗️ 1. 로그인 아키텍처 구현 방향 (2가지 옵션)

현재 백엔드가 서버리스 환경(Cloudflare Workers)이므로, 인증(Authentication)을 처리하는 데에는 두 가지 주요 전략이 있습니다. 

### 💡 옵션 A: 인증형 서비스(BaaS / Auth Provider) 도입 (⭐ 적극 추천)
**Supabase Auth**, **Clerk**, 또는 **Firebase Auth**와 같은 인증 전문 외부 서비스를 도입하는 방식입니다.

*   **장점 (학생 설명용 포인트):**
    *   **"바퀴를 다시 발명하지 마라":** 소셜 로그인(구글, 페이스북)의 복잡한 OAuth 설정과 보안 토큰 관리를 서비스가 알아서 해줍니다.
    *   **보안성 입증:** 비밀번호 암호화, 휴면 계정 처리, 비밀번호 찾기 이메일 발송 등 민감하고 귀찮은 작업을 완벽하게 대행합니다.
    *   **개발 속도 압도적 단축:** 프론트엔드에 몇 줄의 코드만 넣으면 즉시 3가지 로그인이 연동됩니다. (개발 공수 80% 감소)
*   **단점:** 외부 서비스에 의존적이며, 사용자가 매우 많아질 경우 유료 플랜이 필요할 수 있습니다. (초기 트래픽은 100% 무료)
*   **추천 서비스:** 현재 DB가 Postgres(NeonDB)이므로, 강력한 인증 기능을 제공하는 **Supabase Auth**를 가장 추천합니다.

### 🛠️ 옵션 B: JWT 기반 100% 자체 인증 시스템 구축 (Cloudflare Workers + NeonDB)
구글, 페이스북의 개발자 센터에서 직접 API 키를 발급받고, 이메일/비밀번호 로그인을 위한 보안 처리(Bcrypt 등)와 JWT(JSON Web Token) 발급을 Workers 서버에서 직접 구현하는 방식입니다.

*   **장점 (학생 설명용 포인트):**
    *   **완벽한 자유:** 데이터베이스 구조부터 세션 유지 기간까지 모든 것을 100% 입맛대로 통제할 수 있습니다.
    *   **학습 효과 극대화:** 학생들에게 OAuth 2.0의 원리, JWT의 작동 방식, 비밀번호 단방향 암호화 처리 과정을 바닥부터 가르칠 수 있는 최고의 실습 시나리오입니다.
*   **단점:** 구현해야 할 비즈니스 로직(토큰 갱신, 보안 예외 처리, 각 소셜 플랫폼별 API 변동 대응)이 매우 많고 까다롭습니다.

> **👨‍🏫 안티그라비티의 제안:** **실무 현장의 트렌드와 보안 안전성을 고려할 때 "옵션 A (Supabase Auth 등)"를 사용하는 것을 강력히 제안**합니다. 만약 교육 목적상 '구축의 본질'을 보여주셔야 한다면 "옵션 B"로 진행해도 충분히 멋진 결과물을 낼 수 있습니다.

## 🎨 2. 프론트엔드 UI/UX 기획 제안

프리미엄 바버샵의 브랜드 이미지를 해치지 않으면서 직관적인 로그인 화면을 설계해야 합니다.

### 모달(Modal) 형태의 우아한 진입
별도의 로그인 페이지로 화면이 전환되는 것보다, 기존 랜딩 페이지의 프리미엄 느낌을 유지한 채 **부드럽게 떠오르는 모달(레이어 팝업)** 형태를 제안합니다. 예약(Book Now) 버튼을 눌렀을 때 비로그인 상태라면 자연스럽게 나타나도록 유도합니다.

### 액션 버튼의 시각적 계층 (Visual Hierarchy)
버튼의 배치 순서는 사용성이 가장 좋은 **"소셜 로그인 우선"** 배치 트렌드를 따릅니다.

1.  **[로고 영역]** 중앙에 우아한 금장 K 로고 배치
2.  **구글로 계속하기 (Continue with Google):**
    *   디자인: 하얀색 바탕 + 구글 G 로고 바탕 + 가장 큰 크기 (가장 많이 쓰임)
3.  **페이스북으로 계속하기 (Continue with Facebook):**
    *   디자인: 페이스북의 딥블루(#1877F2) 바탕 + 흰색 폰트
4.  --- (또는 이메일로 로그인하기) --- 형태의 구분선
5.  **이메일 / 비밀번호 입력 폼:**
    *   디자인: 어두운 다크 테마 인풋 박스
6.  **계정 만들기 (회원가입) / 비밀번호 찾기** 링크 텍스트

## 🔒 3. 연동 준비를 위한 필수 사전 세팅 (진행 확정 시 필요)

코딩에 돌입하기 전, 다음 항목들의 승인이 필요합니다.

1.  **Google Cloud Console:** OAuth 2.0 클라이언트 ID 및 보안 키 발급 필요
2.  **Facebook Developers:** 페이스북 로그인 앱 생성 및 App ID, Secret 발급 필요
3.  **인증 서버 선택:** 옵션 A(BaaS) vs 옵션 B(자체 구축) 중 하나를 결정해주세요.
