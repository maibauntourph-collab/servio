## 1. 왜 SNS 로그인인가?

| 구분 | 일반 이메일 가입 | SNS 로그인 |
|---|---|---|
| 가입 소요시간 | 2~3분 (이메일, 비밀번호, 인증 등) | **3초** (버튼 1번 탭) |
| 이탈률 | 약 60~70% | **10~20%** |
| 비밀번호 분실 | 자주 발생 | 없음 |
| 본인인증 | 이메일 인증 필요 | SNS가 대신 검증 |

> **핵심:** 바버샵 고객은 "예약"이 목적이지, 복잡한 회원가입은 원치 않습니다. 
> **SNS 원클릭 로그인 = 예약 전환율 극대화**

---

## 2. 추천 SNS 로그인 4가지 비교

| SNS | 주요 타겟 | 장점 | 단점 |
|---|---|---|---|
| **Google** | 전 세계 | 안드로이드 기본 계정, 가장 넓은 커버리지 | 없음 |
| **Kakao** | 한국인 고객 | 국내 점유율 1위, 카카오톡 알림 연동 가능 | 해외 유저 사용 불가 |
| **Facebook** | 필리핀·동남아 고객 | 필리핀 국민 앱, 현지 고객 필수 | 젊은층 사용 감소 추세 |
| **Apple** | 아이폰 유저 | iOS 앱스토어 필수 요건, 프라이버시 강점 | 안드로이드 사용 불가 |

### 🏆 K-Barber 추천 조합
> **Google + Facebook + Kakao** (3종)
> - Google: 글로벌 기본
> - Facebook: 필리핀 현지 고객 확보 핵심
> - Kakao: 한국인 고객 전용 채널

## 3. 사용자 경험 (UX) 플로우

```
[고객이 "예약하기" 클릭]
        ↓
┌─────────────────────────┐
│    로그인 / 회원가입      │
│                         │
│  🔵 Google로 계속하기    │
│  🟡 카카오로 계속하기    │
│  🔷 Facebook으로 계속하기│
│                         │
│  ─── 또는 ───           │
│  📧 이메일로 가입하기    │
└─────────────────────────┘
        ↓
[SNS 버튼 클릭 → 3초 만에 로그인 완료]
        ↓
[바로 예약 화면으로 이동]
```

### 핵심 원칙
- **원클릭 가입:** 버튼 하나로 가입과 로그인을 동시에 처리
- **추가 정보 입력 없음:** 이름, 이메일은 SNS에서 자동으로 가져옴
- **자동 프로필 생성:** 첫 로그인 시 DB에 고객 프로필 자동 생성

## 4. 기술 아키텍처 (개요)

```
[사용자 브라우저]
      ↓ SNS 버튼 클릭
[Supabase Auth] ← OAuth 2.0 → [Google / Kakao / Facebook]
      ↓ 인증 토큰 발급
[Hono 백엔드 (Cloudflare Workers)]
      ↓ 유저 정보 저장
[NeonDB (PostgreSQL)]
      ↓
[프론트엔드에 세션 쿠키 전달 → 로그인 상태 유지]
```

### Supabase Auth를 쓰는 이유
- **무료 티어:** 월 50,000명까지 무료
- **OAuth 내장:** Google/Facebook/Kakao 등 설정만 하면 바로 사용
- **JWT 토큰:** 보안이 검증된 표준 인증 방식
- **Row Level Security:** 데이터베이스 레벨에서 접근 제어
