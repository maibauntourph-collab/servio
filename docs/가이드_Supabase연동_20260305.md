# 👨‍🏫 Supabase 로그인 시스템 최종 적용 가이드 (교수님 전용)

안녕하세요 교수님! 옴니채널 로그인(구글, 페이스북, 이메일)을 위한 모든 코드 구현을 마쳤습니다. 
이제 실제 작동을 위해 **Supabase 대시보드에서 가져온 키값들을 다음 위치에 딱 한 번만 넣어주시면 됩니다.**

---

## 1. 프론트엔드 설정 (Frontend)
파일 위치: `e:\HAIRSHOP\frontend\.env` (없으면 `.env.example`을 복사해서 생성해 주세요)

아래 두 항목을 Supabase Project Settings > API 에서 복사해 넣어주세요.
```env
VITE_SUPABASE_URL=당신의_프로젝트_URL
VITE_SUPABASE_ANON_KEY=당신의_ANON_KEY
```

## 2. 백엔드 설정 (Backend)
파일 위치: `e:\HAIRSHOP\backend\wrangler.toml`

아래 항목을 Supabase Project Settings > API > **JWT Settings**에 있는 **JWT Secret** 값으로 채워주세요.
```toml
[vars]
SUPABASE_JWT_SECRET = "여기에_Supabase_JWT_Secret_입력"
```

## 3. 소셜 로그인 활성화 (Supabase Dashboard)
학생들에게 보여주실 때 아래 경로에서 버튼만 켜주시면 즉시 작동합니다!
- **Google:** Supabase > Auth > Providers > Google (Enabled ✅)
- **Facebook:** Supabase > Auth > Providers > Facebook (Enabled ✅)

---

### 💡 학생들을 위한 핵심 설명 포인트 (교수님 참고용)
1. **"신분증(JWT)의 원리":** 사용자가 구글로 로그인하면 Supabase가 신분증(토큰)을 줍니다.
2. **"검문소(Middleware)":** 백엔드 서버는 `supabaseAuth` 미들웨어를 통해 이 신분증이 위조되지 않았는지(Secret 일치 여부)를 매번 확인합니다.
3. **"데이터 무결성":** 이제 '누가' 예약했는지를 Supabase의 고유 ID(UUID)를 통해 정확히 추적할 수 있습니다.
