# QR 코드 어둡게 보이는 원인 분석 보고서

## 📊 이미지 파일 상태 비교표

| 파일명 | 용량 | 상태 | 비고 |
|---|---|---|---|
| `premium-qr-original-A.png` | **2,019 KB** | ✅ 정상 (고해상도) | 밝은 금색 프레임 |
| `premium-qr-original-B.png` | **1,909 KB** | ✅ 정상 (고해상도) | 밝은 금색 프레임 |
| `premium-qr-inal-C.png` | **6 KB** | ⚠️ 비정상 (극소) | 깨진/샘플 파일 가능성 |

## 🔍 원인 1: D 이미지 자체가 어두운 색조

D 이미지(`premium-qr-original-D.png`)는 **짙은 남색~검정 배경 위에 금색 QR 패턴**이 그려진 디자인입니다.
A와 B는 **밝은 금색 프레임**이 주를 이루지만, D는 **어두운 메탈릭 톤**이 대부분을 차지합니다.

웹사이트의 배경색도 `#0A0A0A`(거의 검정)이므로, **어두운 이미지 + 어두운 웹 배경 = 거의 안 보임** 현상이 발생합니다.

## 🔍 원인 2: C 이미지는 깨진 파일 (6KB)

`premium-qr-inal-C.png`의 용량은 단 **6KB**입니다. A(2MB), B(1.9MB)에 비교하면 **300배 이상 작습니다.**
이것은 고해상도 QR 프레임 이미지가 아니라 **깨진 파일이거나 아주 작은 샘플 이미지**입니다.
또한 파일명에 오타가 있습니다: `premium-qr-inal-C` (❌ `orig` 누락)

## 🔍 원인 3: 이 이미지들은 "장식용 아트"이지 실제 스캔 가능한 QR이 아님

A, B, C, D 이미지는 모두 **AI가 생성한 장식용 3D 렌더링**입니다.
이 안에 그려진 QR 패턴은 실제로 스캔하면 작동하지 않는 **가짜(눈속임) QR 무늬**입니다.
실제 스캔 가능한 QR은 별도로 `premium-qr-install.png` (7KB, 흑백)으로 존재합니다.

## 💡 해결 방안 (3가지 옵션)

### 방안 1: D 이미지를 밝은 버전으로 교체
교수님께서 밝은 색조의 D 디자인을 별도로 준비해 교체하는 방법

### 방안 2: CSS 밝기 필터 적용
코드에서 D 이미지만 자동으로 밝게 보정:
```css
/* D 이미지 선택 시 밝기를 1.3배로 올림 */
filter: brightness(1.3);
```

### 방안 3: AI로 밝은 D 이미지 새로 생성
제가 `generate_image` 도구로 A/B와 비슷한 밝기의 새 D 디자인을 만들어 교체
